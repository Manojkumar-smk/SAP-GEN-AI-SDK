ğŸ“ 4. Templates & Placeholders

Placeholder Syntax Rules

  • Syntax: {{?variable_name}} â€” double curly brace + ? + name.
  • Fill at runtime using TemplateValue(name="x", value="...").
  • {{ ?grounding_response }} is AUTO-FILLED by grounding module â€” never fill it manually.
  • Unused placeholders cause a runtime error â€” always provide all values.
  • SystemMessage cannot have placeholders â€” only UserMessage does.

Template Code Pattern

# Define template once
template = Template(messages=[
  SystemMessage("You are an SAP expert."),
  UserMessage("Answer: {{?question}}")
])

# Fill at runtime
service.run(
  config=config,
  template_values=[
    TemplateValue(
      name="question",
      value="What is SAP BTP?"
    )
  ]
)

Streaming Pattern

    if chunk.orchestration_result:
        delta = chunk.orchestration_result.choices[0].delta.content or ""

ğŸ’¬ 9. Multi-Turn Chatbot Pattern
âš ï¸
LLMs are stateless. They have NO memory between calls. You must manually send the entire conversation history with every request.

How History Works

TurnWhat you send
Turn 1[UserMessage("What is SAP BTP?")] â†’ get reply
Turn 2[UserMessage("BTP?"), AssistantMsg("...reply..."), UserMessage("Tell me more")]
Turn 3Full history of Turns 1 + 2 + new message

Chatbot Key One-Liners

  • Pass messages_history=self.history to .run() â€” the SDK prepends these before template messages.
  • After each turn, append both UserMessage(input) AND the reply_message object to history.
  • Store the full Message object (not just the text) â€” the SDK needs role + content.
  • History grows with every turn â€” for long conversations, trim older messages to avoid exceeding token limits.
  • Call bot.reset() to clear history and start a fresh conversation.

ğŸ¯ 12. Interview Q&A â€” Flash Cards
Q1: What is the current SAP Gen AI SDK package name and why did it change?
The new package is sap-ai-sdk-gen. The old generative-ai-hub-sdk was deprecated in 2024 as SAP standardized the SDK naming across their AI product portfolio.
Q2: What does OrchestrationService do and how do you initialize it?
It's the main client that executes your AI pipeline against SAP AI Core. Initialize with just the deployment URL â€” OrchestrationService(api_url="..."). Credentials are auto-read from ~/.aicore/config.json.
Q3: What is OrchestrationConfig and what can it hold?
It's the pipeline blueprint. Required: template + llm. Optional: grounding, data_masking, filtering. All passed as keyword arguments to OrchestrationConfig().
Q4: What is the template placeholder syntax in SAP Orchestration?
{{?variable_name}} â€” double curly brace + question mark + name. Fill at runtime with TemplateValue(name="variable_name", value="...") in the template_values list.
Q5: Explain RAG and how grounding implements it in SAP SDK.
RAG = Retrieve relevant document chunks, then generate an answer from them. In SAP SDK, GroundingModule runs a similarity search on a knowledge base, injects retrieved text into {{ ?grounding_response }}, and the LLM answers from that context.
Q6: What are the two grounding source types and when do you use each?
help.sap.com â€” for official SAP product documentation, no setup needed. DataRepositoryType.VECTOR â€” for your own documents indexed in SAP AI Core's Document Grounding Service, requires a data_repositories ID.
Q7: What is the difference between Anonymization and Pseudonymization?
Anonymization is irreversible â€” "John Smith" â†’ "[PERSON]". Pseudonymization is reversible â€” "John Smith" â†’ "ENTITY_001", and the SDK can map tokens back. Use anonymization for GDPR compliance, pseudonymization when you need original values post-processing.
Q8: What are the content filter threshold values and what do they mean?
0 = block all (strictest), 2 = allow low severity, 4 = allow medium, 6 = allow most (permissive). Applies to 4 categories: hate, self_harm, sexual, violence. Enterprise default is 0.
Q9: How do you implement a multi-turn chatbot using the Orchestration Service?
Maintain a history list of Message objects. On each call pass messages_history=self.history to .run(). After each turn append both the UserMessage and the assistant's reply object to history.
Q10: What is the difference between .run() and .stream()?
.run() waits for the complete response and returns it at once â€” best for APIs and batch jobs. .stream() returns a generator of partial chunks â€” best for chat UIs where real-time token display improves UX. Access delta via chunk.orchestration_result.choices[0].delta.content.
Q11: What are the correct LangChain imports for 2024+ and what changed?
Use langchain_community.vectorstores (not langchain.vectorstores), langchain_text_splitters (not langchain.text_splitter), and LCEL chains with | instead of deprecated RetrievalQA. Chat model and embeddings come from gen_ai_hub.proxy.langchain.openai.
Q12: How do you inspect the retrieved grounding chunks in the response?
result.module_results.grounding.data['grounding_result'] â€” useful for debugging, showing sources to users, or building citation features.
Q13: What is LCEL (LangChain Expression Language)?
A composability syntax that chains LangChain components using the | pipe operator â€” like Unix pipes. Example: retriever | prompt | llm | StrOutputParser(). Replaces older chain classes like RetrievalQA.
Q14: What is the order in which modules execute in the pipeline?
Data Masking â†’ Prompt Template â†’ Grounding â†’ LLM â†’ Content Filter. Masking always runs first so PII is never sent to grounding or LLM. Content filter always runs last to check the LLM output before returning to the user.
